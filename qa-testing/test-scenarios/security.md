# Тестовые сценарии: Security (Безопасность)

## Обзор
Критические тесты безопасности для проверки защиты приватных ключей, шифрования и защиты от атак.

---

## SEC-001: Шифрование seed-фразы при хранении

**Приоритет:** Критический  
**Тип:** Безопасность данных

**Шаги:**
1. Создать новый кошелёк
2. Закрыть приложение
3. Найти файл хранилища (electron-store)
4. Проверить содержимое файла

**Ожидаемый результат:**
- Seed-фраза НЕ хранится в открытом виде
- Данные зашифрованы AES-256-GCM
- Без мастер-пароля расшифровка невозможна

---

## SEC-002: Защита приватных ключей в памяти

**Приоритет:** Критический  

**Шаги:**
1. Разблокировать кошелёк
2. Сделать дамп памяти процесса
3. Поискать seed-фразу или приватные ключи в дампе

**Ожидаемый результат:**
- Приватные ключи минимальное время в памяти
- После использования — очистка памяти
- Seed-фраза не найдена в открытом виде

---

## SEC-003: Brute-force защита пароля

**Приоритет:** Критический  

**Шаги:**
1. Заблокировать кошелёк
2. Попытаться разблокировать с неправильным паролем 5+ раз
3. Измерить время между попытками

**Ожидаемый результат:**
- Argon2id замедляет проверку (минимум 500мс на попытку)
- После 5 неудачных попыток — увеличение задержки
- После 10 попыток — временная блокировка (5 мин)

---

## SEC-004: Автоматическая блокировка по таймауту

**Приоритет:** Высокий  

**Шаги:**
1. Разблокировать кошелёк
2. Не совершать никаких действий
3. Подождать время таймаута (по умолчанию 5 мин)

**Ожидаемый результат:**
- Кошелёк автоматически заблокирован
- Требуется повторный ввод пароля
- Приватные ключи очищены из памяти

---

## SEC-005: Блокировка при сворачивании

**Приоритет:** Средний  

**Шаги:**
1. Разблокировать кошелёк
2. Свернуть приложение
3. Подождать 1 минуту
4. Развернуть приложение

**Ожидаемый результат:**
- Кошелёк заблокирован (если включено в настройках)
- Экран блокировки отображается

---

## SEC-006: Валидация адресов перед отправкой

**Приоритет:** Критический  

**Шаги:**
1. Перейти в "Отправить"
2. Ввести невалидный адрес:
   - Неправильная длина
   - Неправильная контрольная сумма
   - Адрес другой сети (ETH адрес в BTC поле)

**Ожидаемый результат:**
- Ошибка валидации для каждого случая
- Транзакция не создаётся
- Чёткое сообщение об ошибке

---

## SEC-007: Подтверждение крупных транзакций

**Приоритет:** Высокий  

**Шаги:**
1. Попытаться отправить сумму > 50% баланса
2. Проверить процесс подтверждения

**Ожидаемый результат:**
- Дополнительное предупреждение о крупной сумме
- Требуется двойное подтверждение
- Показана информация о получателе

---

## SEC-008: Защита от clipboard-атак

**Приоритет:** Высокий  

**Шаги:**
1. Скопировать адрес получателя в буфер
2. Вставить в поле отправки
3. Проверить соответствие

**Ожидаемый результат:**
- Приложение предупреждает о вставке из буфера
- Предложение проверить адрес визуально
- Сравнение первых и последних символов

---

## SEC-009: Экспорт seed-фразы

**Приоритет:** Критический  

**Шаги:**
1. Перейти в Настройки → Безопасность
2. Нажать "Показать seed-фразу"
3. Ввести мастер-пароль
4. Просмотреть seed-фразу

**Ожидаемый результат:**
- Требуется ввод пароля
- Предупреждение о безопасности перед показом
- Seed-фраза показывается на ограниченное время
- Кнопка "Скрыть" / автоскрытие через 60 сек
- Запрет скриншотов (если возможно на платформе)

---

## SEC-010: Смена мастер-пароля

**Приоритет:** Высокий  

**Шаги:**
1. Перейти в Настройки → Безопасность
2. Нажать "Сменить пароль"
3. Ввести текущий пароль
4. Ввести новый пароль (дважды)
5. Подтвердить

**Ожидаемый результат:**
- Проверка текущего пароля
- Валидация нового пароля (минимум 8 символов)
- Данные перешифрованы новым паролем
- Успешная разблокировка новым паролем

---

## SEC-011: Защита от подмены приложения

**Приоритет:** Высокий  

**Шаги:**
1. Проверить подпись исполняемого файла
2. Проверить хеш-суммы

**Ожидаемый результат:**
- Исполняемый файл подписан (Windows/Mac)
- Хеш-суммы совпадают с официальными

---

## SEC-012: Безопасность IPC (Inter-Process Communication)

**Приоритет:** Критический  

**Шаги:**
1. Проверить preload.ts на exposed APIs
2. Попытаться вызвать неразрешённые методы из renderer

**Ожидаемый результат:**
- Только whitelist API доступен renderer процессу
- contextIsolation включён
- nodeIntegration отключён
- Нет прямого доступа к Node.js из renderer

---

## SEC-013: Защита сетевых запросов

**Приоритет:** Высокий  

**Шаги:**
1. Перехватить сетевой трафик (Wireshark/Charles)
2. Проверить запросы к API

**Ожидаемый результат:**
- Все запросы через HTTPS
- Нет утечки приватных данных
- Certificate pinning (если реализовано)

---

## SEC-014: Ghost Mode — анонимность

**Приоритет:** Критический  

**Шаги:**
1. Активировать Ghost Mode
2. Конвертировать BTC → XMR
3. Проверить связываемость транзакций

**Ожидаемый результат:**
- Входящая BTC транзакция не связана с выходящей XMR
- Используются разные адреса
- Нет metadata связывающей транзакции

---

## SEC-015: Очистка данных при удалении кошелька

**Приоритет:** Высокий  

**Шаги:**
1. Создать кошелёк
2. Перейти в Настройки → "Удалить кошелёк"
3. Подтвердить удаление
4. Проверить файловую систему

**Ожидаемый результат:**
- Все данные кошелька удалены
- Файлы хранилища очищены
- Невозможно восстановить без seed-фразы

---

## SEC-016: SQL/NoSQL Injection (если применимо)

**Приоритет:** Высокий  

**Шаги:**
1. В любое текстовое поле ввести:
   - `'; DROP TABLE wallets; --`
   - `{"$gt": ""}`
2. Отправить форму

**Ожидаемый результат:**
- Ввод обработан как текст
- Нет ошибок базы данных
- Данные не повреждены

---

## SEC-017: XSS защита в UI

**Приоритет:** Средний  

**Шаги:**
1. В поле заметки к транзакции ввести:
   - `<script>alert('xss')</script>`
   - `<img src=x onerror=alert('xss')>`
2. Сохранить и просмотреть

**Ожидаемый результат:**
- Скрипты не выполняются
- HTML экранирован
- Контент отображается как текст

---

## SEC-018: Резервное копирование

**Приоритет:** Высокий  

**Шаги:**
1. Создать резервную копию кошелька
2. Проверить содержимое файла бэкапа

**Ожидаемый результат:**
- Бэкап зашифрован
- Seed-фраза не в открытом виде
- Для восстановления нужен пароль

---

## SEC-019: Проверка обновлений

**Приоритет:** Средний  

**Шаги:**
1. Проверить механизм обновлений
2. Попытаться подсунуть фейковое обновление

**Ожидаемый результат:**
- Обновления только с официального источника
- Проверка подписи обновления
- Предупреждение при несоответствии

---

## SEC-020: Логирование и аудит

**Приоритет:** Средний  

**Шаги:**
1. Проверить файлы логов приложения
2. Поискать чувствительные данные

**Ожидаемый результат:**
- Пароли не логируются
- Приватные ключи не логируются
- Seed-фразы не логируются
- Только необходимая информация для отладки
